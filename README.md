# EternalReturn_Stat

이터널리턴 실험체 스탯 비교 도구. **"방어력 1 = 체력 N"** 같은 단순 환산을 넘어,
**캐릭터가 실제로 쓰는 스탯 맥락 안에서 아이템 효율(DPS·EHP)을 비교**하는 것이 목표.

> 예: *"이 캐릭터는 공격력이 낮은 대신 공속이 높네 → 그럼 어떤 아이템이 더 좋지?"* 에 정확한 수치로 답한다.

## 핵심 설계

분석 단위는 캐릭터가 아니라 **(캐릭터 × 무기군) = 빌드 프로파일**.
같은 캐릭터라도 무기군에 따라 평타 계수와 메인 스탯이 바뀌기 때문.

스탯을 3개 라인으로 분류해 각각 하나의 "결과값"으로 수렴시킨다:

| 라인 | 스탯 | 결과값 |
|------|------|--------|
| 공격 | 공격력·스킬증폭·공속·치확/치피·방관·쿨감 | **유효 DPS** (캐릭터 계수 적용) |
| 생존 | 체력·방어·회복·흡혈 | **유효 체력(EHP)** + 초당 회복 |
| 유틸 | 이속·방해저항 | 별도 표기 (환산 불가) |

**아이템 비교 = 한계효율**: 아이템 장착 전후의 DPS·EHP 변화량으로 순위를 매긴다.
→ 같은 아이템도 빌드에 따라 순위가 달라진다 (`npm run demo`로 확인).

## 프로젝트 구조

```
src/
  types.ts       # 데이터 스키마 (Character, WeaponBuild, Skill, BuildProfile ...)
  constants.ts   # 게임 공식 상수/공식 — ⚠️ // VERIFY 표기값은 검증 후 교체
  engine.ts      # 순수 함수 계산 엔진 (DPS/EHP/한계효율)
  engine.test.ts # 엔진 테스트
  demo.ts        # 빌드별 아이템 효율 순위 데모
data/
  sampleCharacters.ts  # 샘플 데이터 (⚠️ // SAMPLE — 실밸런스 아님)
```

## 실행

```bash
npm install
npm test         # 엔진 테스트
npm run demo     # 빌드별 아이템 효율 순위 출력
npm run typecheck
```

## ⚠️ 데이터/공식 검증 필요 항목

엔진의 **구조**는 완성됐지만, 정확한 게임 수치는 아직 샘플/임시값이다.

- `src/constants.ts`의 `DEFENSE_K`, `ATTACK_SPEED_CAP`, `CDR_CAP`, 관통/치명타 적용 방식 → 인게임 실측·공식 데이터로 검증
- `data/sampleCharacters.ts`의 모든 `// SAMPLE` 수치 → 실데이터로 교체
