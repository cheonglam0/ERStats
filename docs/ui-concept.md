# UI 구상안 — 상용화를 위한 디자인 기반 구축

> 목적: 지금의 "브라우저 기본 UI로 만든 개인 툴"에서 **자체 비주얼 아이덴티티를 가진 상용 서비스**로.
> 이 문서는 **0순위(UI 기반)** 의 설계도이며, 이후 랜딩·공유카드·빌드추천이 모두 이 위에 얹힌다.
> 작성: 2026-06-19

## 0. 전제 — 왜 지금 "강점이 없어 보이는가"

코드를 뜯어보면 기능은 충실한데, **시각 요소가 전부 브라우저 기본값**이다. 이게 아마추어 신호의 정체다.

| 현재 | 위치 | 문제 |
|------|------|------|
| 아이콘이 **이모지** (`✕` `🔗` `＋` `★`) | [App.tsx:452](../src/ui/App.tsx#L452), `.remove`, `.skill-badge` | OS마다 다르게 렌더, 정렬·크기 제각각 → 가장 큰 아마추어 신호 |
| 빌드 저장이 **`window.prompt()`** | [App.tsx:222](../src/ui/App.tsx#L222) | 브라우저 기본 회색 팝업. "직접 만든 제품" 느낌 0 |
| **시스템 폰트** ("Malgun Gothic") | [styles.css:20](../src/ui/styles.css#L20) | 한글이 흐리멍덩. 상용 한국 사이트는 100% 웹폰트 |
| **기본 `range`/`checkbox`** | 전역 | 슬라이더·체크박스가 OS 위젯 그대로 노출 |
| 평면 색·그림자 없음 | [styles.css:1](../src/ui/styles.css#L1) `:root` | 토큰은 깔끔하나 "엑셀 느낌". 깊이·발광·질감 없음 |
| 로고·브랜드 없음 | [App.tsx:293](../src/ui/App.tsx#L293) | `<h1>` 텍스트뿐 |

> 핵심 통찰: **CSS 구조 자체는 좋다**(시맨틱 토큰, 색 변수, 반응형). 갈아엎을 게 아니라, **그 위에 ‘디자인 시스템 + 컴포넌트 + 에셋’ 세 층을 얹으면** 된다.

## 1. 디자인 원칙

이터널리턴은 **루미아 섬의 SF 배틀로얄 + 애니풍**. 인게임 UI는 다크 베이스에 시안/블루 발광. 이 톤을 웹으로 가져온다.

1. **Data-dense but calm** — 숫자가 많은 사이트. 색은 의미(공격=레드/생존=블루/유틸=퍼플)에만 쓰고 나머지는 무채색.
2. **Dark glass** — 패널은 반투명 글래스 + 미세 발광 보더. 평면 카드 탈피.
3. **One identity color** — 게임 톤에 맞춘 **시안~블루 그라데이션**을 1차 브랜드 색으로 (현재 노란 `--accent`는 강조 포인트로 격하/공존).
4. **No emoji, no native popups** — 전부 SVG 아이콘 + 자체 컴포넌트.
5. **3초 룰** — 첫 화면에서 "여기서 뭘 하는지"가 즉시 읽혀야 함.

## 2. 도입할 UI 기반 (스택)

원칙: **Vite+React 그대로, 가볍게, 기존 CSS 토큰을 살리는 방향.** 무거운 풀-프레임워크 교체는 리스크 대비 효과가 낮다.

### 2-1. 추천 스택 (권장)

| 층 | 채택 | 이유 / 비용 |
|----|------|------------|
| **폰트** | **Pretendard** (self-host, OFL 무료) | 단일 변경으로 체감 품질 최대. 토스·당근 등 상용 한국 사이트 표준. 영문/숫자도 우수 |
| **아이콘** | **lucide-react** (~)  | 이모지 전량 교체. tree-shaking, SVG, 일관된 stroke. `npm i lucide-react` |
| **스타일 토큰** | **기존 CSS 변수 확장** (Tailwind 미도입) | 이미 잘 짜인 `:root` 토큰을 *gradient/glow/elevation/radius/motion* 까지 확장. 학습비용·rewrite 0 |
| **컴포넌트** | **자체 primitives** (`src/ui/kit/`) | Button·Card·Tab·Badge·Tooltip·Slider·Modal·Toast. 외부 라이브러리 의존 최소화 |
| **모션** | **CSS transition + 소수 keyframes** | framer-motion은 추천 추후 필요 시. 지금은 CSS로 충분 |

> 합계 추가 의존성: **lucide-react 1개**(+폰트 파일). 번들 영향 미미.

### 2-2. 대안 (참고)

- **Tailwind + shadcn/ui**: 업계 표준, 컴포넌트 즉시 확보. 단 776줄 기존 CSS와 충돌·이중관리, 학습/마이그레이션 비용 큼. → *지금 단계엔 과함.*
- **Mantine / Chakra**: 컴포넌트 풍부하나 디자인이 "그 라이브러리 티"가 남 → 차별화 목표와 역행.

**결론: 2-1 추천 스택.** 의존성 최소 + 기존 자산 보존 + 100% 우리 디자인.

## 3. 비주얼 아이덴티티

### 3-1. 브랜드 네이밍 — **확정: Aglumia (아글루미아)**
인게임 연구조직 **Aglaia(아글라이아)** + 무대 **Lumia(루미아 섬)** 의 융합.
- 의미: "아글라이아가 루미아 섬에서 실험체를 연구·계측한다" → 본 사이트의 정체(빌드 계측)와 정확히 일치.
- 표기: 영문 **Aglumia** / 한글 **아글루미아** / 도메인 **aglumia.gg** (확보 가능 시)
- 대안(보류): `Lumiaglaia`(두 단어 완전보존·길다), `Aglaumia`
- 로고: 워드마크 `◈ Aglumia` — 심볼 ◈ 는 미스릴 결정(다이아) 모티프, 미스릴 시안 그라데이션.

### 3-2. 컬러 시스템 — **이터널리턴 5대 특수재료 팔레트**
게임의 엔드 재료 색을 그대로 브랜드 팔레트로. 기존 시맨틱 색과 거의 1:1로 대응돼 교체가 자연스럽다.

| 재료 | 색 | 토큰 | 역할 |
|------|----|------|------|
| **미스릴** (청록·은빛) | `#22d3ee` | `--mithril` | **1차 브랜드 / 링크 / 포커스** |
| **포스 코어** (금·호박) | `#fbbf24` | `--forcecore` | **강조 · 최고효율**(기존 `--accent` 대체) |
| **VF 혈액** (진홍) | `#f43f5e` | `--blood` | **공격 DPS**(기존 `--offense`) |
| **생명의 나무** (에메랄드) | `#34d399` | `--tree` | **생존·회복**(EHP/힐) |
| **운석** (자수정 보라) | `#a78bfa` | `--meteor` | **유틸·스킬증폭**(기존 `--utility`) |

```css
/* 기존 베이스 유지 */
--bg:#0f1320  --panel:#181d2e  --line:#2c3450  --text:#e6e9f2  --muted:#8a92ad

/* 5대 재료 = 브랜드 팔레트 (기존 시맨틱 색을 이 값으로 정렬) */
--mithril:#22d3ee;  --forcecore:#fbbf24;  --blood:#f43f5e;
--tree:#34d399;     --meteor:#a78bfa;

/* 시맨틱 별칭 (기존 변수명 유지 → 점진 교체 무중단) */
--brand:var(--mithril);   --accent:var(--forcecore);
--offense:var(--blood);   --survival:#4dabf7/*HP블루 유지*/;  --utility:var(--meteor);

/* 깊이·발광·모션 */
--grad-brand: linear-gradient(135deg,#22d3ee,#3b82f6);   /* 미스릴→블루(SF톤) */
--glass: rgba(24,29,46,.6);
--glow-brand: 0 0 24px rgba(34,211,238,.25);
--elev-1: 0 1px 2px rgba(0,0,0,.4);   --elev-2: 0 8px 24px rgba(0,0,0,.45);
--radius:12px;  --radius-sm:8px;
--ease: cubic-bezier(.2,.8,.2,1);  --dur:160ms;
```
> 미스릴 시안을 1차 브랜드로 두면 게임 인게임 UI(시안 발광)와 톤이 맞고, 나머지 4색이 데이터 의미색으로 분담된다.

### 3-3. 타이포 스케일
Pretendard 기준. `font-variant-numeric: tabular-nums`는 수치 전역 유지(이미 일부 적용됨).
```
display 28/700 · h1 20/700 · h2 15/700 · body 14/400 · sm 12 · xs 11
```

### 3-4. 질감
- 패널: `var(--glass)` + `backdrop-filter: blur(8px)` + 1px 발광 보더
- 활성 탭/선택 카드: 브랜드 그라데이션 보더 + `--glow-brand`
- hover: 150ms `--ease` 트랜지션 (현재 즉시 전환 → 부드럽게)

## 4. 컴포넌트 시스템 (`src/ui/kit/`)

기본 HTML/이모지를 대체할 최소 primitive 세트. 각각 작은 파일 1개.

| 컴포넌트 | 대체 대상 (현재) |
|----------|------------------|
| `Button` (variant: primary/ghost/danger) | `.build-actions button`, `.tab`, 이모지 버튼들 |
| `Icon` (lucide 래퍼) | `✕`(X) `🔗`(Link2) `＋`(Plus) `★`(Star) `🔍`(Search) |
| `Card` / `GlassPanel` | `.panel`, `.card` |
| `Tabs` / `SegmentedControl` | `.view-tabs`, `.weapon-tabs`, `.metric-toggle` |
| `Badge` / `Tag` | `.skill-badge`, `.tier-badge`, `.dmg-tag`, 등급칩 |
| `Slider` | 기본 `range` (레벨·숙련도·방어력) |
| `Tooltip` | 현재 `title=""` 속성 (호버 즉답으로 승격) |
| `Modal` / `Dialog` | `window.prompt`(빌드 저장) 대체 |
| `Toast` | "복사됨 ✓" 인라인 텍스트 대체 |

> 전부 무의존(또는 lucide만). 기존 클래스명을 점진 교체 가능하게 설계.

## 5. 화면 리디자인 방향

### 5-1. 앱 셸 — 상단 네비 (현 `.topbar` 교체)
지금: 제목 텍스트 + 알약 탭 9개 평면 나열. → **로고 + 그룹화된 네비 + 검색.**

```
┌──────────────────────────────────────────────────────────────┐
│ ◈ Loadout   [티어] [빌드 계산기] [비교] [도감] [패치]   🔍검색  │  ← 9탭 → 5그룹
└──────────────────────────────────────────────────────────────┘
   (스탯/스킬/아이템/정보 = '빌드 계산기' 하위 세그먼트로 통합)
```
- 9개 평면 탭([App.tsx:103](../src/ui/App.tsx#L103))을 **5개 상위 + 하위 세그먼트**로 재편 → 정보구조 명확화.

### 5-2. 랜딩(티어 뷰) — 히어로 추가
첫 방문자가 가치를 즉시 인지하도록 티어표 위에 히어로 1블록:
```
┌──────────────────────────────────────────────┐
│   "내 빌드에서 어떤 아이템이 진짜 더 셀까?"     │  ← 한 줄 가치 제안
│   실험체 × 무기군 빌드의 DPS·EHP를 수치로.      │
│   [ 빌드 계산기 시작 → ]   [ 인기 빌드 보기 ]   │  ← 명확한 CTA
└──────────────────────────────────────────────┘
       ↓ 아래로 dak.gg 기반 티어표(현행 유지·스타일만 업)
```

### 5-3. 빌드 결과 카드 (현 `.result-cards` 강화)
DPS/EHP 숫자 카드에 **글래스 + 발광 + 미니 추세**. 공유카드(다음 단계 OG 이미지)의 시각 원형이 됨.

### 5-4. 디테일 교체 목록 (빠른 효과)
- 이모지 → lucide 아이콘 전량
- `window.prompt` → `Modal` (빌드 이름 입력)
- "복사됨 ✓" → `Toast`
- 기본 슬라이더 → 브랜드색 커스텀 트랙
- 등급/티어 배지 → 발광 보더 `Badge`
